# 扩大地图网格设计规格

**日期：** 2026-06-23  
**状态：** 已批准  
**关联规格：** `docs/superpowers/specs/2025-06-23-bomberman-design.md`

---

## 1. 概述

### 目标

将默认对战地图从 **13×11** 扩大到 **21×17**，增加可玩面积，缓解「格子太少、场地偏小」的感受。

### 成功标准

- 地图正确渲染为 21 列 × 17 行，硬墙布局（外圈 + 棋盘格柱）合法。
- 地图在 960×720 窗口内居中显示，HUD 不遮挡游戏区域。
- 四角出生、安全区、软墙散布、连通性校验均正常工作。
- 移动、放弹、爆炸、道具、AI 寻路等现有玩法无回归。
- 主设计文档中的网格尺寸引用已同步更新。

### 不在范围内

- 放大单格像素（`TILE_SIZE` 保持 32）
- 修改窗口尺寸（保持 960×720）
- 调整软墙密度（保持 55%）
- 设置界面增加地图尺寸选项
- 多张地图模板或多尺寸预设
- 调整 AI 参数、炸弹/火力上限等玩法数值

---

## 2. 方案选择

采用**方案 1：修改 `GameConstants` 常量**。

仅将 `GRID_WIDTH` / `GRID_HEIGHT` 改为 21 / 17。出生点、`MAP_PIXEL_SIZE`、`map_offset()` 均由常量推导，地图生成与渲染逻辑无需改动。

未采用「设置可切换地图尺寸」或「独立 MapConfig 资源」，因超出当前需求且违反 YAGNI。

---

## 3. 常量变更

**文件：** `scripts/constants.gd`

| 常量 | 旧值 | 新值 |
|------|------|------|
| `GRID_WIDTH` | 13 | **21** |
| `GRID_HEIGHT` | 11 | **17** |
| `TILE_SIZE` | 32 | 32（不变） |
| `MAP_PIXEL_SIZE` | 416×352 | **672×544**（自动推导） |
| `WINDOW_SIZE` | 960×720 | 960×720（不变） |
| `SOFT_WALL_DENSITY` | 0.55 | 0.55（不变） |

**出生点**（`SPAWN_CELLS`，由宽高推导，无需手改）：

| 角色 | 坐标 |
|------|------|
| 玩家 | `(1, 1)` |
| AI 1 | `(19, 1)` |
| AI 2 | `(1, 15)` |
| AI 3 | `(19, 15)` |

**布局边距**（`map_offset()` 自动计算）：

- 水平：`(960 - 672) / 2 = 144px`
- 垂直：约 88px（含 HUD 上方 20px 偏移）

---

## 4. 地图生成

`MapGenerator` 逻辑不变，以下规则在 21×17 下自动适配：

| 规则 | 说明 |
|------|------|
| 外圈硬墙 | 围成 21×17 边界 |
| 棋盘格硬墙柱 | `x % 2 == 0 && y % 2 == 0`；宽高均为奇数，布局合法 |
| 软墙散布 | 内圈 19×15 范围，密度 55% |
| 安全区 | 各出生点曼哈顿距离 ≤ 2 内不生成软墙 |
| 连通性校验 | BFS 确保所有活跃出生点互通，最多重试 10 次 |

---

## 5. 玩法与系统影响

以下参数**全部不变**：

- 炸弹引信 2.5 秒、火力上限 8、炸弹数量上限 8
- 移动速度档位、道具掉落率 20%
- 身体接触、穿弹规则
- AI 三档难度参数
- 窗口尺寸与单格像素

地图变大后对战空间更开阔，属于预期效果，不额外调参。

**AI 寻路：** BFS 在 357 格（21×17）上运行，性能无压力。
