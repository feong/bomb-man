# 玩家移动手感优化设计

**日期：** 2026-06-24  
**状态：** 已批准  
**范围：** 仅玩家输入与表现；AI 与网格逻辑规则不变  

---

## 1. 背景与问题

当前玩家在 `player.gd` 每帧调用 `try_move()`，而 `Bomber.try_move()` 在 `_is_moving == true` 时直接返回，导致移动过程中的方向输入被丢弃。格心到达时若仍按着某方向键，角色会继续沿该方向移动。

**典型症状：**

- 快速左右连按时，中间按键无效，体感「不跟手」
- 松手后角色仍沿最后按住的方向滑行

**约束：**

- 保持四方向离散网格移动（格心 lerp），不改为半格自由移动
- 炸弹仍放置在单元格内（移动中放弹使用 `_previous_cell`）
- 仅优化玩家；AI 保持现有逐步 `try_move` 逻辑

---

## 2. 目标

| 目标 | 说明 |
|------|------|
| 即时动画跟手 | 方向输入立刻更新朝向与 walk 动画，不等待 `try_move` 成功 |
| 逻辑格权威 | `grid_pos` 仅在格心结算时按规则变更；快速连按可只改朝向、不改位置 |
| 无延迟滑行 | 不用多步输入队列；松手后最多走完当前格即停 |
| 经典拐弯 | 移动中提前按垂直方向，到格心自动拐弯（若可走） |

---

## 3. 方案选择

在以下方案中选定 **方案 C（输入缓冲 + 格心结算）+ 表现与位移分离**：

| 方案 | 描述 | 结论 |
|------|------|------|
| A | 经典炸弹人：仅格心转向，无额外机制 | 无法解决输入丢弃 |
| B | 半格中途改向/折返 | 与格子权威、放弹占格冲突，不采用 |
| **C** | 单槽缓冲 + 格心消费 + 即时朝向动画 | **采用** |

实现落点：**方案 1**——表现与缓冲逻辑集中在 `Player`，`Bomber` 仅新增 `set_facing()`。

---

## 4. 架构：双层分离

```
┌─────────────────────────────────────┐
│  Player 输入层                       │
│  - 读方向（最后按下优先）              │
│  - set_facing() → 即时动画           │
│  - 单槽缓冲 _buffered_dir            │
│  - 格心结算 _on_move_finished()      │
└──────────────┬──────────────────────┘
               │ try_move(dir) 仅在格心/静止时
               ▼
┌─────────────────────────────────────┐
│  Bomber 逻辑层（不变）                 │
│  - grid_pos、lerp、碰撞、放弹格        │
└─────────────────────────────────────┘
```

- **表现层**：每帧由当前输入驱动，可独立于位移
- **逻辑层**：`try_move` 仍控制 `grid_pos` 与 lerp；AI 不经过输入层

---

## 5. 即时朝向与动画

### 5.1 `Bomber.set_facing(dir: Vector2i)`

新增公开方法，仅更新 `_facing` 并调用现有 `_play_walk(dir)` 逻辑，**不移动、不修改 `grid_pos`**。

### 5.2 玩家每帧行为

有方向输入时，**先于** `try_move` 调用 `set_facing(dir)`。

无方向输入且未在移动时，调用 `_play_idle()`。

### 5.3 原地快速连按

站在格心快速左右（或任意方向）切换：

- 朝向与动画立即跟随最后一次输入
- `grid_pos` 不变
- 不播放完整原地走路循环（仅切换朝向 / walk 动画状态），避免「原地小跑」

### 5.4 移动中换向

lerp 进行中按下新方向：

- 朝向立刻更新（可能出现极短「位移方向与面朝方向不一致」）
- 位移仍沿当前格 lerp 直至格心
- 格心后按缓冲与按住规则决定是否折返或拐弯

---

## 6. 单槽缓冲规则

| 规则 | 说明 |
|------|------|
| 容量 | 仅 1 个 `_buffered_dir`，不排队 |
| 写入 | `_is_moving` 且 `dir != ZERO` → 覆盖 `_buffered_dir` |
| 清空 | 四方向键全部松开 → `_buffered_dir = ZERO` |
| 不写缓冲 | 静止时直接 `try_move(dir)` |

不设多步队列，不设超时（松手清空即可避免延迟滑行）。

---

## 7. 每帧处理顺序（`Player._physics_process`）

1. 读取当前方向 `dir`（见第 8 节）
2. 若 `dir != ZERO` → `set_facing(dir)`
3. 若 `dir == ZERO` → 清空 `_buffered_dir`；若未移动则 `idle`
4. 若未在移动 → `try_move(dir)`（`dir == ZERO` 则跳过）
5. 若在移动且 `dir != ZERO` → `_buffered_dir = dir`
6. 检测移动结束边沿（上一帧 `_is_moving`，本帧否）→ `_on_move_finished()`

---

## 8. 输入优先级：最后按下优先

替换现有固定 `if/elif`（上 > 下 > 左 > 右）：

- 用 `Input.is_action_just_pressed` 更新 `_last_dir`
- 当前仍按着的方向中，以 `_last_dir` 为准（若该键仍按住）
- 同时按两轴（如 W+D）时，以**最后按下**的轴为准

---

## 9. 格心结算（`_on_move_finished`）

当前格 lerp 完成时：

1. 读取当前按住的方向 `held_dir`
2. 若 `held_dir != ZERO` → `try_move(held_dir)`
3. 否则若 `_buffered_dir != ZERO` → `try_move(_buffered_dir)`，然后清空缓冲
4. 否则保持 idle（第一节：松手后走完当前格即停）

`held_dir` 优先于缓冲。移动中 tap 后已松手时缓冲已清空，不会在格心多走。

### 9.1 典型用例

| 场景 | 表现 | 逻辑位置 |
|------|------|----------|
| 原地快速左右 | 朝向跟手切换 | 不变 |
| 右移中 tap 左后松手 | 朝向变左；走完当前格停 | 完成进行中的右移一格后停止，不折返 |
| 右移中按住左至格心 | 朝向左；格心折返 | 折返向左一格 |
| 右移中提前按住下至格心 | 朝向下；格心拐弯 | 向下移动 |
| 移动中放炸弹 | — | 仍在离开的格（`_previous_cell`） |

---

## 10. 不变项

- `try_move` / `_finish_move` 内占格、拾取、同格阵亡逻辑不变
- `get_bomb_placement_cell()` 不变
- `GameConstants` 移动时长与速度档位不变
- `AIController` 不接入缓冲与 `set_facing`

## 12. 测试计划

1. **原地连按**：快速左右，朝向跟手，`grid_pos` 不变  
2. **tap 反向后松手**：右移中 tap 左后松手，走完当前格即停，不多滑  
3. **按住折返**：右移中按住左，格心向左折返（可走时）  
4. **路口拐弯**：水平移动中提前按住垂直方向，格心拐弯  
5. **撞墙**：顶墙时朝向更新，位置不变  
6. **移动中放弹**：炸弹落在离开的格子上  
7. **AI 回归**：AI 移动、放弹、躲避行为与改前一致
