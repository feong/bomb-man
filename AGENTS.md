# AGENTS.md — AI 协作指南

本文档为在本仓库中工作的 AI Agent 提供项目级约定。请在与本项目相关的所有任务中遵守以下规则。

## 语言要求

**所有文档必须使用中文撰写。**

适用范围包括但不限于：

- `docs/` 目录下的设计文档、规格说明、实现计划、架构说明
- `README.md` 及项目根目录下的其他 Markdown 文档
- 代码内面向开发者的说明性注释（业务逻辑注释可用中文；API 名称、类名、函数名保持英文）
- Commit 说明（若由 Agent 撰写）
- PR 描述与测试计划（若由 Agent 撰写）
- 与用户的对话回复（默认使用中文）

### 例外

以下情况可保留英文，无需强行翻译：

- 源代码中的标识符（变量名、类名、函数名、文件名）
- Godot 引擎 API、GDScript 关键字
- 第三方库名、协议名、URL
- Git 分支名（若项目已有英文命名惯例）

### 文档存放位置

| 类型 | 路径 |
|------|------|
| 设计规格 | `docs/superpowers/specs/` |
| 实现计划 | `docs/superpowers/plans/` |
| 其他项目文档 | `docs/` |

新建文档时，文件名可使用英文（便于工具链识别），但**正文内容必须为中文**。

## 项目概述

基于 **Godot 4.x + GDScript** 的 2D 炸弹人游戏（单人 vs AI）。

- 设计规格：`docs/superpowers/specs/2025-06-23-bomberman-design.md`
- 玩法、架构、UI 等细节以该规格为准

## 开发约定

1. **先读规格再动手** — 实现前阅读最新设计文档，有冲突时以文档为准或先更新文档。
2. **最小改动** — 只改与当前任务相关的代码，不做无关重构。
3. **遵循现有结构** — 目录与命名参考设计文档第 8 节「项目文件结构」。
4. **占位美术可换皮** — 逻辑与资源路径分离，便于后续替换素材。
5. **游戏内 UI 文案** — 全部使用中文（见设计规格第 5 节）。

## 技术栈

- 引擎：Godot 4.x（最新稳定版）
- 语言：GDScript（带类型标注）
- 渲染：Compatibility
- 目标平台：桌面（Windows / macOS / Linux）
- 窗口：960×720
